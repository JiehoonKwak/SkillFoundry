# Assets

This directory belongs to the experiment skill `cellphonedb-analysis`.

The current refine pass upgrades the package from a static contract to a tiny deterministic CellPhoneDB-style starter. The assets remain synthetic and should not be interpreted biologically.

## What the toy assets validate

- The runnable contract between `examples/`, `scripts/`, `metadata.yaml`, and the declared deliverables.
- Group-level expression summaries, a bounded label-shuffle null, DEG relevance flags, and adjacency support written as machine-readable QC artifacts.
- Repeatable ranking behavior that a general shell-capable coding agent can regenerate locally in a few seconds.

## What the toy assets do not validate

- The full public CellPhoneDB database, heteromer handling, or large-iteration statistical workflow.
- Real DEG calling, orthology conversion, or atlas-backed microenvironment definitions.
- Biological significance beyond the deterministic synthetic starter contract.

## Starter boundary

- Truly computed: group means, expression fractions, interaction means, empirical p-values from a tiny shuffle null, DEG support flags, and adjacency support.
- Approximated: DEG mode is reduced to a supplied toy relevance table and interaction ranking is a lightweight heuristic instead of the full CellPhoneDB v5 scoring stack.
- Surrogate: the ligand-receptor catalog, DEG calls, and graph are synthetic starter inputs only.

## Upgrade path

- Replace the toy input with a pinned real counts matrix, metadata table, DEG table, and optionally microenvironment restrictions.
- Record the exact CellPhoneDB release, database snapshot, thresholds, and permutation depth in `cellphonedb_report.md` and `refs.md`.
- Keep the same deliverable names and schemas when promoting the experiment to a stronger runnable starter.
