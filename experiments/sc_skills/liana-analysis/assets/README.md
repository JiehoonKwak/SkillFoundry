# Assets

This directory belongs to the experiment skill `liana-analysis`.

The current refine pass upgrades the package from a static contract to a tiny deterministic LIANA-style starter. The assets remain synthetic and should not be interpreted biologically.

## What the toy assets validate

- The runnable contract between `examples/`, `scripts/`, `metadata.yaml`, and the declared deliverables.
- Metadata harmonization, group-level expression summaries, explicit method scores, a bounded shuffle null, condition deltas, and adjacency support written as machine-readable QC artifacts.
- Repeatable LIANA-like ranking behavior that a general shell-capable coding agent can regenerate locally in a few seconds.

## What the toy assets do not validate

- The full LIANA or LIANA+ package stack, robust rank aggregation implementation, or large-scale permutation workflow.
- Real OmniPath resource snapshots, curated complexes, spatial autocorrelation statistics, or tensor decomposition.
- Biological significance beyond the deterministic synthetic starter contract.

## Starter boundary

- Truly computed: metadata harmonization, group means, expression fractions, min-expression and geometric-mean interaction scores, a specificity heuristic, empirical p-values from a tiny shuffle null, condition deltas, resource overlap counts, and adjacency support.
- Approximated: rank aggregation is a simple mean-rank starter and the shuffle null is intentionally tiny instead of a production LIANA or CellPhoneDB run.
- Surrogate: the ligand-receptor catalog, condition labels, and adjacency graph are synthetic starter inputs only.

## Upgrade path

- Replace the toy input with a pinned real counts matrix, metadata table, resource snapshot, and optionally a spatial neighborhood graph.
- Record the exact LIANA or LIANA+ release, selected methods, thresholds, resource version, and permutation depth in `liana_report.md` and `refs.md`.
- Keep the same deliverable names and schemas when promoting the experiment to a stronger runnable starter.
